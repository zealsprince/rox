# rox docs

The rox spec, split into three layers by altitude plus a research track. Product says
what, why, and for whom. Architecture says what the parts are and where the lines
between them run. Implementation is real and runnable: schemas, formats, sequences.
Start with the [Architecture Overview](02-architecture/01-overview.md) for the system
picture, or [Vision](01-product/01-vision.md) for the reasoning behind it.

- [Product](01-product/) - vision, experience, and scope. Start at vision.
- [Architecture](02-architecture/01-overview.md) - the overview carries the system
  diagram and the ADR index; components and the non-functional model hang off it.
- [Implementation](03-implementation/README.md) - the per-domain doc set; each gets
  written when its detail is real, not before.
- [Research](0R-research/README.md) - explorations that need a standalone prototype
  before any decision leans on them.
